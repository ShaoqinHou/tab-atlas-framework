# TabAtlas Product Lead Skill

Use this skill when changing product behavior, UX flows, or categorization semantics.

## Core product truth

The user is using tabs as memory. The goal is not to shame or force cleanup; the goal is to make the pile navigable and safe to close later.

## Good UX decisions

- Show small meaningful groups before long lists.
- Make every category explainable.
- Let the user refine with natural language.
- Keep raw fallback data available.
- Never force a rigid taxonomy.

## Bad UX decisions

- Defaulting to a 1,000-row table.
- Creating generic folders like "Technology" when specific intent is visible.
- Hiding low-confidence uncertainty.
- Treating browser tab groups as final truth.
- Automatically closing or moving tabs.

## Review checklist

- Can a non-technical user understand this screen?
- Does the UI reduce anxiety?
- Is there a safe preview before destructive actions?
- Does it work even if Codex is unavailable?
